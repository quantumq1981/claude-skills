# TITLE: Anti-Slop Frontend Layout and Design Audit

## Core Design Rules
1. **Ban the AI Defaults**: Completely forbid the "premium consumer palette" (e.g., #FAF9F6, #1A1A1A, soft beige). Use custom, striking color values tailored to the project brief.
2. **Zero Em-Dashes**: Do not use em-dashes (—) in headlines, hero text, buttons, or navigation blocks. 
3. **No Triple Feature Cards**: Break up lists of features. Never default to three equal, side-by-side icon-and-text grid boxes. Use asymmetric layouts or variable component sizing.
4. **Audit First**: Before writing any new frontend code, scan the existing files. List the spacing inconsistencies, weak typographic hierarchy, and generic patterns.
5. **No Placeholders**: Ship 100% complete, fully implemented code. Do not leave `// TODO: add styles here` or generic placeholder names.
